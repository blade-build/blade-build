# 构建产物与输出

## 终端显示特性

Blade 为终端输出提供了以下增强：

- **彩色输出：** 构建过程采用高亮彩色展示，便于视觉区分
- **错误高亮：** 错误信息以醒目颜色标识，便于快速定位
- **精简输出：** 命令执行细节默认隐藏，仅在出错时显示，避免刷屏
- **日志文件记录：** 所有输出同时写入 `blade-bin/blade.log`，便于完整追溯

## 生成的产物

### 默认构建行为

- **架构：** 默认生成与本机架构一致的可执行文件
- **架构选择：** 通过 `-m32` / `-m64` 选项指定 32 位或 64 位目标
- **构建模式：** 默认 release 构建，使用 `-p debug` 可生成 debug 版本
- **作用范围：** 默认只构建当前目录下的目标
- **依赖处理：** 需要时自动重新构建依赖的外部模块

### 递归构建

递归构建当前目录及其所有子目录下的目标：

```bash
blade build ...
```

## 构建输出组织

### 关注点分离

Blade 在构建产物与源代码之间保持严格隔离：

- **独立的输出目录：** 不同构建配置生成到不同的输出目录
- **源代码保护：** 生成的文件与源代码目录隔离，避免污染
- **配置隔离：** 不同构建选项组合拥有各自独立的输出空间

## 构建产物管理

### 清理操作

清理构建结果（得益于增量构建的高效性，通常无需频繁清理）：

```bash
blade clean
```

### 主要收益

- **增量高效：** 极少需要执行全量清理
- **空间优化：** 智能依赖追踪避免冗余构建
- **配置安全：** 多种构建配置并存而互不干扰
- **开发效率：** 通过有选择的重新构建显著缩短构建时间
