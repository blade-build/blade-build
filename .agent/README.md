# `.agent/` — AI agent working area

This directory is a convention for artifacts produced by AI coding
agents (Claude, Cursor, Copilot, etc.) while working on this repo.
It is **not** part of the project's tracked source tree — only a few
explicitly whitelisted paths (see [`.gitignore`](../.gitignore)) are
committed. Everything else is local-only and safe to delete at any
time.

## Layout

| Subdirectory | Purpose | Tracked in Git? | Lifecycle |
| --- | --- | --- | --- |
| `scratchpad/` | Free-form notes, pseudo-code, stream-of-thought drafts. | No | Throw away anytime. |
| `artifacts/` | Generated outputs an agent produced for the user (exported docs, images, one-off reports). | No | Auto-clean; regenerable on demand. |
| `tools/` | Agent-authored helper scripts specific to this repo (Python / shell). | **Yes**, with review. | See [`tools/README.md`](tools/README.md). |
| `context/` | Project indexes, knowledge-base caches, embeddings. | No | Rebuilt automatically by the agent. |

The directory shape is advisory — subdirectories are created on
demand, not pre-seeded. If you open a fresh clone and `.agent/` is
empty except for this README and `tools/`, that is expected.

## Why not just `.gitignore` the whole thing?

Because agent-generated helper scripts sometimes have genuine
cross-session reuse value (audit sweepers, doc-style checkers, release
aids). Burying them in a gitignored directory wastes that value;
committing them to the project's existing `scripts/` directory mixes
them with hand-written core tooling and blurs provenance. A separate
tracked `.agent/tools/` keeps the "this was written by an agent and
reviewed before landing" signal explicit.

## Relationship to `scripts/`

- `scripts/` (if/when this repo has one) = human-authored core
  tooling that's part of the project's normal lifecycle.
- `.agent/tools/` = helpers generated by AI agents, reviewed and
  kept because they proved useful. Tagged at the directory level
  for provenance; never silently promoted to `scripts/` without a
  dedicated PR.

## For contributors (human)

You can ignore this directory entirely. Nothing in the build, tests,
or release pipeline reads from it. The only paths that ship to a
fresh clone are:

- `.agent/README.md` (this file)
- `.agent/tools/README.md`
- `.agent/tools/*.py`, `.agent/tools/*.sh` (if any)

## For future agents

Route new artifacts using this four-way decision:

1. **Single-use** (PR body, `gh` intermediate, shell pipe) → system
   temp via `mktemp -t <slug>.XXXXXX.<ext>`; do *not* write to
   `.agent/`.
2. **Repo-scoped, reusable, agent-authored** → `.agent/tools/`
   (this repo) with a proper CLI and docstring, then open a
   dedicated review PR.
3. **Cross-project general-purpose** → offer to publish as a GitHub
   Gist; do not drop it here.
4. **Local working notes you want across sessions** →
   `.agent/scratchpad/` or `.agent/artifacts/` or `.agent/context/`
   as appropriate. All three are gitignored.

Cross-repo rationale lives in
[`agent-work-artifacts-layout`](https://github.com/chen3feng/agent-skills/blob/master/skills/agent-work-artifacts-layout/SKILL.md).
