# `.agent/tools/` — agent-authored helpers

Scripts in this directory were generated by AI coding agents while
working on this repo, then reviewed and kept because they proved
genuinely useful. They are **not** part of the project's core build
or release pipeline; nothing in `src/`, `test/`, or CI depends on
them.

See [`../README.md`](../README.md) for the full `.agent/` layout.

## What goes here

- Small repo-scoped Python / shell helpers (audit sweepers,
  doc-style checkers, refactor scripts, log parsers, etc.).
- Only scripts **specific to this repo**. Cross-project utilities
  should be a GitHub Gist instead.

## What does *not* go here

- Project core tooling (release, lint, bootstrap) — those belong in
  a top-level `scripts/` if/when the project grows one, authored
  and maintained by humans.
- Throwaway one-shot scripts — keep them in `.agent/scratchpad/`
  (gitignored) or delete them.
- PR body drafts, `gh` intermediate files — use `mktemp` in the
  system temp dir.

## Intake standard

Every script committed to this directory **must** satisfy all of:

1. **Provenance header.** Top-of-file comment naming the agent
   session intent, e.g.:
   ```python
   # Generated by Claude (2026-04, session: post-merge housekeeping sweep)
   # Purpose: one-line description of what the script does and why it exists.
   ```
2. **Proper CLI.** `argparse` with `--help`; no hard-coded paths
   relative to CWD without a flag to override.
3. **Dry-run by default.** Anything that mutates files, git state,
   or remote services must default to preview and require an
   explicit `--apply` / `--write` flag to commit changes.
4. **Docstring with usage example.** Module-level docstring showing
   at least one real invocation.
5. **Self-contained.** Uses only the stdlib, or dependencies that
   the project already has. No fresh `requirements.txt` entries
   just for an agent helper.
6. **UTF-8 safe.** Explicit `encoding="utf-8"` on file I/O; assume
   non-ASCII inputs.
7. **Dedicated PR.** Introduced (or modified) in its own PR, not
   smuggled into an unrelated feature/bugfix PR.

Scripts that don't meet all seven belong in `.agent/scratchpad/`
(gitignored) instead.

## Tracked file types

The repo's `.gitignore` whitelists only `*.py`, `*.sh`, and this
`README.md` under `tools/`. Auxiliary artifacts (`.pyc`,
`__pycache__/`, editor caches) stay ignored automatically.

## Deprecation

A script here that hasn't been invoked in a while is fair game for
removal — treat this directory as a living toolbox, not an archive.
If something proves to have long-term value, promote it to
`scripts/` with a dedicated PR (new provenance: "originally authored
by agent, audited and graduated").
